<h3 style="color: #059669;">自动驾驶芯片对比分析</h3>

<blockquote style="border-left: 4px solid #059669; padding: 12px 16px; background: #ecfdf5; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：车载芯片的竞争焦点不是谁 TOPS 大，而是每瓦能跑多少 TOPS——闷在密封盒里，谁热降频谁就输了。
</blockquote>

### 主流方案对比

| 芯片 | 制程 | AI 算力 | TDP | 能效比 | 典型客户 |
|------|------|---------|-----|--------|----------|
| `SA8650` | 4nm | ~100 TOPS | 15-25W | ~5 TOPS/W | 中高端 ADAS，你的平台 |
| `SA8255` | 5nm | ~50 TOPS | 10-15W | ~4 TOPS/W | 中端 ADAS |
| `Orin-X` | 8nm | 254 TOPS | 45-60W | ~5 TOPS/W | 蔚来、理想、小鹏 |
| `Orin-Y` | 8nm | 200 TOPS | 35-45W | ~5 TOPS/W | 中端智驾 |
| `Thor-X` | 4nm | 1000 TOPS | ~100W | ~10 TOPS/W | 下一代旗舰 |
| `Mobileye EyeQ6H` | 5nm | 34 TOPS | 20-30W | ~1.4 TOPS/W | 极氪、大众 |
| `Horizon J6E` | 7nm | 80 TOPS | 20-25W | ~3.6 TOPS/W | 比亚迪、理想 L6 |
| `Horizon J6P` | 7nm | 128 TOPS | 25-35W | ~4.3 TOPS/W | 高端智驾 |
| `TI TDA4VH` | 7nm | 32 TOPS | 10-20W | ~2 TOPS/W | 低速泊车、性价比 |

### TOPS 数字是陷阱

芯片 A 标 254 TOPS，芯片 B 标 100 TOPS——不代表 A 比 B 强 2.5 倍：

- **TOPS 是峰值算力**：实测利用率通常只有 30-50%，受带宽、编译器、模型结构影响
- **INT8 TOPS ≠ FP16 TOPS**：NPU 擅长 INT8 推理，GPU 擅长 FP16，直接比数字没意义
- **内存墙**：算力再大，数据搬不过来也是空转——`SA8650` 的 4nm 在带宽上优于 `Orin-X` 的 8nm
- **编译器差异**：高通 QNN 对自家 NPU 的算子覆盖率和优化程度，和 Mobileye 封闭生态完全不同

### 为什么车企不都选最大的？

```
1000 TOPS Thor vs 100 TOPS SA8650

Thor 选它因为：算力够跑视觉 + 激光雷达 + BEV + Occupancy 全套
不选它因为：100W = 疯狂散热 + 电池焦虑 + 空调压缩机抢功率 + 贵

SA8650 选它因为：15W 好散热、够用、便宜、高通生态好
不选它因为：100 TOPS 跑大模型吃力
```

- 10 万的车用 J6E（80 TOPS）——够 L2 基础功能
- 30 万的车用 Orin-X（254 TOPS）——城区 NOA
- Robotaxi 用 Thor（1000 TOPS）——不怕贵不怕费电，追求极致安全

</summary>

<h3 style="color: #7c3aed;">NPU vs GPU</h3>

<blockquote style="border-left: 4px solid #7c3aed; padding: 12px 16px; background: #f5f3ff; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：GPU 是全能运动员什么都能算，NPU 是专项冠军只做矩阵乘加——同功耗下 NPU 快 3-5 倍，但 GPU 能干的活 NPU 不一定能干。
</blockquote>

### 计算原理差异

```
GPU（通用并行）                    NPU（专用脉动阵列）
                          
每个 CUDA Core 独立工作：          数据像流水线流过：
┌──┐ ┌──┐ ┌──┐                   ┌─┬─┬─┬─┐
│a1│×│w1│→│r1│                   │a│×│w│→│ │→ 累加
└──┘ └──┘ └──┘                   │ │ │ │ │ │→ 累加
                                  │ │ │ │ │ │→ 累加
逐指令执行                        └─┴─┴─┴─┘
                                  一个时钟周期完成整行矩阵乘
```

- **GPU**：成百上千个 CUDA Core 各自执行指令，每条取指-译码-执行-写回
- **NPU**：脉动阵列（Systolic Array），数据在阵列中节拍式流动，一次读入一行权重，流过整列数据，出来的就是结果

### 关键对比

| | GPU | NPU |
|------|-----|-----|
| **擅长** | 通用并行计算、渲染 | 矩阵乘法、卷积 |
| **指令灵活性** | 高（任意 CUDA 程序） | 低（只支持确定的算子集合） |
| **峰值算力** | FP16/FP32 为主 | INT8/FP16 为主 |
| **同算力功耗** | 高（~15W 跑 10 TOPS） | 低（~3W 跑 10 TOPS） |
| **编程难度** | CUDA 生态成熟 | 私有 SDK（QNN 等），算子覆盖不全就要回退 CPU |
| **典型代表** | NVIDIA A100/H100、Orin GPU | 高通 HTP、苹果 ANE、华为达芬奇 |

### 为什么车上要两者都有

```
SA8650 内部：
┌──────────────────────────────┐
│ NPU（HTP）    100 TOPS       │ → 跑 目标检测、车道线、YfMiner 推理
│ GPU（Adreno）  ~4 TFLOPS     │ → 跑 渲染、图像预处理、BEV 特征提取
│ DSP            标量/矢量      │ → 跑 信号处理、IMU 滤波、脱敏检测
│ CPU（Kryo）    8 核          │ → 跑 调度、规划、QNX 系统
└──────────────────────────────┘
```

不是替代关系——**NPU 干不了 GPU 所有活，GPU 没 NPU 省电**。厂商把四类计算单元塞进同一颗 SoC，是因为车上每种负载都需要最合适的引擎。

### 量化：NPU 省电的关键

NPU 主力跑 INT8（8 位整数），GPU 主力跑 FP16（16 位浮点）：

```
INT8 矩阵乘 需要的晶体管                    FP16 矩阵乘
─────────────────────────              ─────────────────────────
1 byte × 1 byte = 1 个乘法器             2 byte × 2 byte = 4 个乘法器
功耗 ~1×                               功耗 ~4×
面积 ~1×                               面积 ~4×
同一片硅上能塞 4 倍计算单元              同一片硅上只能塞 1/4
```

QNN 把 FP32 模型量化为 INT8，不是简单截断——它校准每个张量的缩放因子，让 INT8 近似的精度损失 < 1%，但功耗降到 1/4。

### 系统性能分析工具

实际部署场景中，可以使用 `socwatch` 等性能分析工具评估系统整体表现。该工具能够采集 CPU、NPU（HTP）、DSP、DDR 等多种 IP 的性能计数器，通过统计推理任务周期内各组件的利用率和负载占比，帮助定位性能瓶颈和资源浪费。
